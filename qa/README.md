# QA Documentation Templates

This folder contains provisional Markdown templates for the QA documentation that will be added during the QA phase.

These files are not execution results and do not claim completed coverage. They are working templates for future manual validation, automation planning, evidence, defects, and traceability.

## Template Set

| Document | Purpose |
| --- | --- |
| `manual-test-scenarios.md` | Manual scenario templates for feature and RBAC validation. |
| `api-test-cases.md` | API automation candidate templates. |
| `gui-test-cases.md` | GUI automation candidate templates. |
| `defect-reports.md` | Defect report templates. |
| `evidence-records.md` | Evidence record templates for screenshots, reports, logs, and notes. |
| `traceability-matrix.md` | Traceability templates linking requirements, scenarios, defects, and automation. |

## Status Labels

- `Template`: structure prepared, no execution performed.
- `Planned`: intended future QA artifact.
- `In progress`: artifact is being prepared or executed.
- `Completed`: validated and reviewed artifact.

## Naming Convention

Use short, stable IDs:

- `MAN-001` for manual scenarios.
- `API-001` for API test cases.
- `GUI-001` for GUI test cases.
- `DEF-001` for defects.
- `EVD-001` for evidence records.
- `TRC-001` for traceability rows.
